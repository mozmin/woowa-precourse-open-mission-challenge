# 프리코스 오픈 미션: 프리코스 챌린지 🎯

**Woowa Open Mission (문자열 계산기 · 로또 · 자동차 경주 UI)**

배포 주소: https://woowa-precourse-open-mission-challe.vercel.app/

> 1~3주차 동안 콘솔에서만 보였던 미션들을,  
> **콘솔 대신 직관적인 UI로 체험할 수 있도록**
> 피그마로 디자인하고,
> 리액트로 구현하고,
> 깃허브로 배포까지 해보며
> 낯선 도구들을 다뤄보는 “도전”을 해보는 프로젝트

### 문자열 덧셈 계산기 페이지

**기본 기능**

- [ ] 문자열 입력 폼
  - placeholder: `예) 1,2:3`, `//;\n1;2;3`
- [ ] “계산하기” 버튼
- [ ] 결과 출력
  - [ ] 정상 입력 시: `결과: 6` 형태로 합계 표시
  - [ ] 비어있는 문자열 → `0`

**시각화**

- [ ] 기본 구분자(`,`, `:`)와 커스텀 구분자(`//\n` 문법)를 구분하여 파싱
- [ ] 음수 포함 시 에러 처리 + 에러 메시지 노출
- [ ] 잘못된 포맷(연속 구분자 등) 시 에러 메시지 노출
- [ ] “파싱 과정 시각화” 영역
  - [ ] 사용된 구분자
  - [ ] 분리된 숫자 목록
  - [ ] 중간 합계 계산 과정

---

### 로또

**기본 기능**

- [ ] 구입 금액 입력 (`1000원` 단위)
- [ ] “로또 구매” 버튼
- [ ] 발급된 로또 번호 목록 표시

**당첨 결과**

- [ ] 당첨 번호 입력 (`1, 2, 3, 4, 5, 6` 형식)
- [ ] 보너스 번호 입력
- [ ] “결과 확인” 버튼
- [ ] 등수별 통계
  - [ ] 1등 ~ 5등까지 당첨 개수
  - [ ] 총 당첨 금액, 수익률 출력

---

### 자동차 경주 페이지

**기본 기능**

- [ ] 자동차 이름 입력 (`쉼표(,)`로 구분)
- [ ] 시도 횟수 입력
- [ ] “경주 시작” 버튼

**시뮬레이션**

- [ ] 라운드별 로그 출력

  - 예:

    ```text
    1라운드
    pobi : --
    pororo : -
    crong : ---

    2라운드
    ...
    ```

- [ ] 최종 우승자 표시
  - 복수 우승자일 경우 `포비, 뽀로로` 같이 보여주기

**시각화 도전**

- [ ] 진행 상황을 귀여운 아이콘으로 표현

## 배포

- Vercel + GitHub 연동을 통해 프론트엔드 전용 배포 파이프라인을 구성했습니다.
- `main` 브랜치에 push가 발생하면 자동으로 다음 순서로 동작합니다.
  1. `frontend` 디렉터리 기준으로 의존성 설치 (`npm install`)
  2. 빌드 (`npm run build`)
  3. 빌드 결과물(`dist`)을 정적 사이트로 배포

**그 외 추가하거나 UX를 향상 시킬 수 있는 기능이 있다면 지속적으로 업데이트 예정.**

## 테스트 전략

- UI와 비즈니스 로직을 분리하기 위해 각 미션의 핵심 로직을 각각 `calculatorCore.js`, `lottoCore.js`, `racingCore.js`로 모듈화.
- 각 코어 모듈마다 단위 테스트를 작성해 콘솔 미션과 동일한 입·출력 테스트를 진행.
  - `calculatorCore.test.js`
    - 빈 문자열 입력 시 “계산할 값을 입력해 주세요.” 에러를 반환하는지
    - `1,2:3`처럼 콤마/콜론 조합도 합계 6으로 계산하는지
    - 괄호와 사칙연산 우선순위, 연속 연산자 에러 처리 확인
  - `lottoCore.test.js`
    - 1000원 단위 금액 입력 시 예상 장수만큼 티켓을 발행하는지
    - 고정된 티켓·당첨 번호 조합으로 1~2등 카운트, 총 상금, 수익률을 검증
  - `racingCore.test.js`
    - 자동차 이름/시도 입력 파서가 요구 조건을 지키는지
    - deterministic 랜덤 함수를 주입해 라운드 수·우승자 계산을 확인
